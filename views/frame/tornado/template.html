# 2.1 模板

模板

  • tornado自带模板语法，不需要再用其他的模板插件.

  • 模板概念: 模板就是html文件，只是其中加入了模板语法，需要服务器的渲染，才能动态的往页面中填入数据正常显示

  • render模块返回一个html页面，不过那都是固定的页面，固定的数据，但是如果数据是不确定的，是会不断改变的，该怎么做呢？

  • 是否可以先把页面写好，然后预留出固定的位置，在需要的时候再填入数据即可？

  • 模板语法总结

    • {{}}

      • 此符号中放入任意 python 表达式，或者模板中的变量

    • {% ... %} {% end %}

      • 此符号中放入模板中的命令，比如 if 、for 和 while 等
      • 需要注意的是，使用 if 等命令是，需要加上 {% end %}

    除此之外，异常处理 try 也可以在模板中使用，但是这样做会让模板变得像 python 模块一样，因此并不建议大家这么做

    • {# #}

      • 模板中的注释语句，可以让模板中指令不执行

    • 必须掌握

      1. 模板符号 {{、{%、{# 三个符号的作用
      2. 控制语句 {% if %} … {% end %} 的用法
      3. 循环语句 {% for … %} … {% end %} {% while … %} … {% end %} 的用法

# 2.3 模板转义

转义

  • 问题一: 模板可以把数据动态添加到页面，如果添加的值是一段html代码的话，会不会被执行呢？

    答：页面并没有解析，只是当作一个字符串，直接在页面上打印出来.tornado默认是自动的转义，传入的数据都会当作字符串，不会被浏览器解析

  • 模板转义总结

    1. 局部转义

      • raw在局部去转义，使用之后不会转义

    2. escape模块转义

  • extends

    {% block name %} {{ 被重写的变量或表达式 }} {% end %}

被block包裹的代码块在子模板中可以被重写，覆盖父模板中的

  • include

    {% include filename%}

    include 可以导入一些其他的模板文件，一般使用 include 的时候，模板文件中不使用 block 块

{% block body %} 重写内容 {% end %}
{% include ./index.html %}
{% end %}

    1. 继承extend 可以继承父模板，从而节省了大量重复代码，同时也为修改带来了极大的方便，但是需要注意的是一定只能单继承，一个模板只能继承一个模板
